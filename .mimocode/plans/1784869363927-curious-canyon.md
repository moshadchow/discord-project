# Fix Broken `mcp-discord` Git Submodule → Convert to Regular Directory

## Problem
`mcp-discord` is registered as a git submodule (mode `160000` in `git ls-tree HEAD`) but:
1. The `.gitmodules` file is missing from the parent repo — never committed or deleted
2. The remote `https://github.com/moshadchow/mcp-discord.git` no longer exists

This causes `git submodule init/update` to fail and makes git operations confusing.

## Plan: Convert to Regular Directory

### Steps
1. Remove the submodule pointer from git's index (but keep the files on disk):
   ```bash
   git rm --cached mcp-discord
   ```
2. Add the `mcp-discord/` directory as a regular tracked directory:
   ```bash
   git add mcp-discord/
   ```
3. Commit the change:
   ```bash
   git commit -m "Convert mcp-discord from broken submodule to regular directory"
   ```

### Files Modified
- Index/staging — submodule reference removed, files added as regular tracked files
- `git ls-tree HEAD` — `mcp-discord` changes from mode `160000` (submodule) to normal directory entries

### Verification
1. `git status` should show a clean working tree after commit
2. `cd mcp-discord && git status` — the inner `.git/` still exists (its own history); can be left as-is or removed later
3. `git ls-tree HEAD -- mcp-discord` should show normal file entries, not `160000`
4. The user should be able to `cd mcp-discord` and work with files normally
