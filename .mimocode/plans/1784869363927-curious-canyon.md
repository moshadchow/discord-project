# Swagger Documentation Tags — Implementation Plan

## Problem
All endpoints currently share one router with `tags=["Auth"]`. Issue endpoints need `tags=["Messages"]`.

## Approach
Split into two routers in `routes.py`:
- `auth_router` with `tags=["Auth"]` — auth endpoints
- `issue_router` with `tags=["Messages"]` — issue endpoints

Both included in `__init__.py` with the same prefix.

### Files to Modify

1. **`api/routes.py`** — Split `router` into `auth_router` and `issue_router`, move each endpoint to its respective router
2. **`api/__init__.py`** — Import and include both routers

### Verification
- Start server, check `/api/docs` — Auth and Messages as separate groups
- Run all tests to confirm no regressions
