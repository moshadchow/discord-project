# Plan: Add `parent_message_id` to the `issues` Table

## Objective

Track reply threads by storing `parent_message_id` — the `discord_message_id` of the original issue that a reply references. Original issues have `NULL`; replies always point to their parent.

---

## Files to Modify

| File | Change |
|------|--------|
| `src/discord_mcp/issues_repository.py` | Add field to `IssueRecord`, add column in `ensure_schema()`, add to INSERT |
| `src/discord_mcp/api/models.py` | Add `parent_message_id` to `IssueRow` SQLModel |
| `src/discord_mcp/api/service.py` | Set `parent_message_id` in `reply_to_issue()`, add to `_row_to_dict()`, add to reply response |
| `src/discord_mcp/api/schemas.py` | Add `parent_message_id` to `IssueResponse` |
| `tests/test_api_service.py` | Update `FakeIssueRow` defaults, add test for parent_message_id in reply |
| `tests/test_api_routes.py` | Update `FakeIssueRow` defaults |

Place after `discord_message_id` for logical grouping.

### 1b. Add column in `ensure_schema()` (after line 82)

```python

    """
    ALTER TABLE issues
    ADD COLUMN IF NOT EXISTS parent_message_id TEXT

)

    """
    CREATE INDEX IF NOT EXISTS idx_issues_parent_message_id
    ON issues(parent_message_id)
    """
)
```
