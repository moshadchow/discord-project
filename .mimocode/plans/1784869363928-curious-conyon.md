# User Authentication System — Implementation Plan

## Overview

Implement a production-ready JWT-based user authentication system following the existing Repository → Service → API architecture in `mcp-discord/src/discord_mcp/api/`.

---

## New Dependencies

Add to `pyproject.toml`:
- `passlib[bcrypt]` — password hashing
- `python-jose[cryptography]` — JWT token generation/verification

---

## Files to Create/Modify

### 1. Config — `src/discord_mcp/config.py` (modify)

Add `AuthConfig` frozen dataclass and `load_auth_config()`:

```python
@dataclass(frozen=True)
class AuthConfig:
    jwt_secret_key: str
    jwt_algorithm: str
    jwt_access_token_expire_minutes: int
```

Load from env vars: `JWT_SECRET_KEY`, `JWT_ALGORITHM` (default `HS256`), `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` (default `1440`).

### 2. Model — `src/discord_mcp/api/models.py` (modify)

Add `UserRow(SQLModel, table=True)`:

| Column | Type | Notes |
|--------|------|-------|
| id | `Optional[int]` PK | `default=None, primary_key=True` |
| username | `str` | `unique=True, index=True, max_length=100` |
| password_hash | `str` | `sa_type=TEXT` |
| full_name | `str` | `max_length=200` |
| email | `Optional[str]` | `unique=True, max_length=255, default=None` |
| role | `str` | `default="User", max_length=50` |
| is_active | `bool` | `default=True` |
| last_login_at | `Optional[datetime]` | `default=None` |
| created_at | `datetime` | `sa_default=func.now()` |
| updated_at | `datetime` | `sa_default=func.now(), sa_onupdate=func.now()` |

Follow existing `IssueRow` pattern: `table=True`, explicit `__tablename__ = "users"`, `Optional[int]` PK.

### 3. Schema — `src/discord_mcp/api/schemas.py` (modify)

Add Pydantic models following existing `*Request` / `*Response` + `success: bool` pattern:

- `LoginRequest(BaseModel)` — `username: str`, `password: str`
- `LoginResponse(BaseModel)` — `success: bool = True`, `access_token: str`, `token_type: str = "bearer"`, `expires_in: int`, `user: UserSummary`
- `UserSummary(BaseModel)` — `id: int`, `username: str`, `full_name: str`, `role: str`
- `LogoutResponse(BaseModel)` — `success: bool = True`, `message: str`
- Reuse existing `ErrorResponse` for errors

### 4. Security Helpers — `src/discord_mcp/core/security.py` (create new)

Module-level helpers:
- `hash_password(password: str) -> str` — bcrypt via passlib
- `verify_password(password: str, hashed_password: str) -> bool`

### 5. JWT Helpers — `src/discord_mcp/core/jwt.py` (create new)

Module-level helpers:
- `create_access_token(user_id: int, username: str, role: str) -> str` — encode with HS256, expiry from config
- `decode_access_token(token: str) -> dict` — decode and validate, raise on expiry/invalid

Create `src/discord_mcp/core/__init__.py` as empty package.

### 6. Repository — `src/discord_mcp/api/repository.py` (modify)

Add `UserRepository` class following existing `IssueRepository` pattern:
- `__init__(self, session: AsyncSession)` with private `_session`
- `get_by_username(username: str) -> Optional[UserRow]`
- `create_user(row: UserRow) -> UserRow` — add, commit, refresh
- `update_last_login(user_id: int) -> None` — update timestamp, commit
- `get_by_id(user_id: int) -> Optional[UserRow]`

### 7. Service — `src/discord_mcp/api/service.py` (modify)

Add `AuthService` class following existing `IssueQueryService` pattern:
- `__init__(self, repository: UserRepository)`
- `authenticate_user(username: str, password: str) -> tuple[Optional[dict], Optional[str]]` — lookup user, verify password, check active, update last_login, return (user_dict, token) or (None, error_message)
- `get_current_user_from_token(token: str) -> Optional[dict]` — decode JWT, lookup user by ID, return dict or None
- Private `_user_to_dict(row: UserRow) -> dict` helper (matches existing `_row_to_dict` pattern)

Structured logging: `logger.info("Login attempt", extra={"username": ...})`, `logger.warning("Failed login", extra={...})`.

### 8. Dependencies — `src/discord_mcp/api/deps.py` (modify)

Add providers following existing chain pattern:
- `get_user_repository(session: AsyncSession = Depends(get_session)) -> UserRepository`
- `get_auth_service(repository: UserRepository = Depends(get_user_repository)) -> AuthService`
- `get_current_user(token: str = Depends(oauth2_scheme), auth_service: AuthService = Depends(get_auth_service)) -> dict` — FastAPI dependency for protected routes; raises `HTTPException(401)` on failure

Add `oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/api/auth/login")` at module level.

### 9. Routes — `src/discord_mcp/api/routes.py` (modify)

Add auth endpoints:

- `POST /api/auth/login` — accepts `LoginRequest`, calls `auth_service.authenticate_user()`, returns `LoginResponse` or 401
- `POST /api/auth/logout` — returns `LogoutResponse` (stateless JWT, frontend removes token)

Protected endpoint example:
- `GET /api/auth/profile` — uses `Depends(get_current_user)`, returns `UserSummary`

Follow existing pattern: `Depends(get_auth_service)`, `HTTPException` for errors, `response_model=` on decorators.

### 10. App Registration — `src/discord_mcp/api/__init__.py` (modify)

Auth routes go on the existing `router` (no separate router needed since prefix `/api` is already set in `create_app()`).

### 11. Database Schema — `src/discord_mcp/api/repository.py` or separate migration

Add `ensure_users_table()` method (raw SQL `CREATE TABLE IF NOT EXISTS users (...)`) following the existing `IssuesRepository.ensure_schema()` pattern with psycopg.

4. **Model** — add `UserRow` to `api/models.py`

9. **Routes** — add auth endpoints to `api/routes.py`

6. Test protected endpoint: `GET /api/auth/profile` with `Authorization: Bearer <token>` header
7. Test unauthorized access: `GET /api/auth/profile` without token → 401
